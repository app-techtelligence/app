## 1. What this project is

TechTelligence is a Brazilian company with three products, sold through one website:

1. **Consultoria (Consultancy)** ‚Äî Data & AI services for companies: implementing data platforms, AI environments, or both. Audience: businesses. B2B.
2. **Curso (Course)** ‚Äî career-transition course for people migrating into IT with no prior experience (Dev, Front-end, Back-end, DevOps, Data Engineer, Data Analyst, Data Scientist, and other roles). Audience: individuals. B2C.
3. **Mentoria (Mentorship)** ‚Äî 1:1 help to land a job or grow in IT: career strategy, LinkedIn optimization, interview prep, skill development. Audience: individuals. B2C.

**Product priority (decided 2026-07-10): consultancy is the initial business focus.** Whenever the three products appear together ‚Äî nav links, cards, dropdowns, prose enumerations, meta tags ‚Äî the order is consultancy ‚Üí course ‚Üí mentorship, in both languages.

Brand narrative that ties everything together: "We build Data & AI solutions for companies, so we know exactly what the market hires for ‚Äî and we train people to get there." This credibility loop appears on Home and About, and as the "talent bridge" band (standout students may be invited to consultancy projects ‚Äî wording is deliberately non-promissory, keep it that way).

## 5. Internationalization (i18n)

- Locales: `pt-BR` (default) and `en`, via `next-intl`, `localePrefix: "as-needed"`, localized pathnames (both apps).
- **Every** UI string lives in `messages/pt-BR.json` and `messages/en.json`. Never hardcode user-facing text. Add both languages in the same commit ‚Äî a vitest **message-parity test** (`apps/web/messages/messages.test.ts`) fails on key drift or empty strings.
- Platform content is bilingual at **two levels**: UI strings via next-intl JSON; DB content via `title_en`/`description_en` columns picked by `lib/content.ts` `localized()` (falls back to PT when EN is missing).
- **Blog posts must be written in BOTH languages** (user decision, 2026-07-10) ‚Äî publishing is blocked until excerpt + body exist in PT and EN. Posts have per-locale slugs (`slug`, `slug_en`); the locale switcher falls back to `/blog` on post pages.
- Signup records the locale in `user_metadata.locale` **from the server URL, not client context** ‚Äî it drives the confirmation-email language.
- SEO (web only): per-locale metadata, `hreflang` alternates, localized OG, sitemap for both locales.

## 5.1 Blog authoring (SEO + covers)

**Traffic goal (2026-07-10): 10k organic visits/day.** The plan lives in `docs/seo/content-plan.md`; `/seo-strategy` maintains it and picks the next posts, `/seo-audit` checks the live site, and the `seo-researcher`/`seo-auditor` agents (`.claude/agents/`) do SERP recon and page checks. Known gap: no Article JSON-LD on posts yet.

Posts live in the Supabase `posts` table ‚Äî authored in the platform admin editor or via **SQL content seeds** in `apps/platform/supabase/content/` (dated files, dollar-quoted; Claude runs them directly via the Supabase MCP ‚Äî see ¬ß8). The `/blog-post` and `/blog-cover` skills encode the full workflow; the essentials:

**SEO conventions (batch #1 proved these):**
- Each post targets **one Google query**; title ‚â§ ~60 chars with click patterns (question + "guia realista", numbered "7 sinais", "guia completo do zero").
- The `excerpt` doubles as the meta description (~140‚Äì170 chars) ‚Äî write it as one.
- Question-phrased H2s (featured snippets), 700‚Äì1100 words per language, 2‚Äì3 internal links between posts (always verify target slugs), GFM tables supported.
- CTAs by audience: B2B ‚Üí `/consultoria` + `/contato`; B2C ‚Üí `/curso` + `/mentoria` (EN: `/en/consulting`, `/en/contact`, `/en/course`, `/en/mentorship`).
- Tags are bilingual (`tags` PT / `tags_en` EN, PT fallback ‚Äî migration 0006). Paired vocabulary: `IA`‚Üî`AI`, `Dados`‚Üî`Data`, `Empresas`‚Üî`Business`, `Carreira`‚Üî`Career`, `TI`‚Üî`IT`, `Investimento`‚Üî`Investment`.

**Covers:**
- Stored in R2 under the `blog/` prefix (`posts.cover_key`); served by the web route `/api/blog-media/[...key]` (content type comes from the R2 object's metadata; 24 h public cache); the cover **is** the post's `og:image`.
- **Text-free, no logo** ‚Äî one cover serves both locales (user decision 2026-07-10 after rejecting geometric brand covers; the generator `apps/web/scripts/generate-blog-covers.mjs` still exists but is not the preferred style).
- Art direction: AI-generated (Gemini) editorial illustrations ‚Äî deep navy `#1A2A44` + amber `#F59E0B` glow, cinematic, negative space, 16:9, people only as silhouettes. Prompt template lives in the `/blog-cover` skill.
- Target **1200√ó630 WebP under ~150 KB**. Social scrapers reject `og:image` > 5 MB (LinkedIn shows "Cannot display preview").
- **Share-preview caches:** social crawlers visit a URL once and cache the result (~a week) ‚Äî a failed scrape keeps showing "no preview" even after the page is fixed. After changing a post's title/excerpt/cover, force a re-scrape: **LinkedIn** ‚Üí Post Inspector (https://www.linkedin.com/post-inspector/); **Facebook/Instagram** ‚Üí Sharing Debugger (https://developers.facebook.com/tools/debug/, "Scrape Again"); **X** ‚Üí no official tool anymore (its Card Validator was retired) ‚Äî share the URL with a throwaway query string (e.g. `?v=2`) to bypass the stale cache. Also a good 10-second pre-flight before sharing anything: confirms title, description and image resolve.
- The admin `CoverUploader` compresses client-side (> 500 KB ‚Üí ‚â§ 1600px WebP q0.82) because the Workers runtime can't run image codecs. To replace an existing cover without touching the DB: `npx wrangler r2 object put "techtelligence-media/<cover_key>" --file <img> --content-type image/webp --remote` (same key keeps `cover_key` valid).

## 8. Build & deploy gotchas (hard-won ‚Äî do not regress)

- **`nodeLinker: hoisted`** in `pnpm-workspace.yaml` ‚Äî OpenNext bundling breaks with pnpm's default symlink layout (workers-sdk#10236). pnpm 11 reads its config from `pnpm-workspace.yaml`, not `.npmrc` (overrides, allowBuilds live there too).
- **`next build --webpack`** in both apps ‚Äî Turbopack chunks fail through the OpenNext adapter. Re-test `pnpm --filter <app> preview` before touching either workaround.
- Keep the **`middleware.ts` filename** ‚Äî OpenNext does not support Next 16's `proxy.ts`.
- Web security headers come from **two sources**: `next.config.ts headers()` (worker-rendered responses) and `public/_headers` generated by the `prebuild` script (asset-served responses). Change them together. Platform only needs `next.config.ts` (all pages dynamic) and its CSP additionally allows `*.supabase.co` (connect-src) and `media-src 'self'`.
- Compat flags: web has `["nodejs_compat", "global_fetch_strictly_public"]` (outbound Turnstile/Resend fetches); platform has `["nodejs_compat"]`.
- Secrets: web uses `.dev.vars` locally + `wrangler secret put` in prod (`TURNSTILE_SECRET_KEY`, `RESEND_API_KEY`). Platform uses `.env.local` with only the **public** Supabase URL + anon key (inlined at build) ‚Äî the service_role key must never appear anywhere in this repo.
- New Supabase migrations must also be appended to `supabase/setup-all.sql` (the one-paste copy for the SQL editor).
- **Supabase: run directly, never ask (user decision 2026-07-12).** Claude applies migrations and SQL straight to the production project (`WebApp` / `mcotsooledmmhmewxduy`) via the claude.ai Supabase MCP (`apply_migration`, `execute_sql`) ‚Äî standing authorization; do not ask for per-migration approval. Order: apply the migration **before** pushing code that reads the new columns (pushes auto-deploy). If the permission classifier blocks the call in auto mode, the durable fix is a user-made allowlist entry in `.claude/settings.json`: `permissions.allow: ["mcp__claude_ai_Supabase"]` (Claude cannot add that rule itself).

## 9. Security (as implemented)

### Marketing site
- Contact pipeline: Zod (shared client/server, honeypot field) ‚Üí Turnstile siteverify (with `cf-connecting-ip`) ‚Üí Resend (replyTo = submitter). Submissions are never stored; PII is never logged. Rate limiting via Cloudflare WAF. Missing `RESEND_API_KEY` locally = log + 200 stub.
- Security headers on every response (CSP, HSTS, nosniff, Referrer-Policy, Permissions-Policy).

### Platform
- **Auth:** Supabase only, never hand-rolled. Email+password with mandatory confirmation; sessions in httpOnly cookies via `@supabase/ssr`; middleware refreshes tokens.
- **Roles:** `profiles.role` ‚àà `student|mentor|admin` (default `student`). Promotion is **SQL/service-role only** ‚Äî no UI (see commented statement in migration `0004_admin.sql`). Users cannot set their own role (column-level grants revoked).
- **Admin enforced in three layers:** Postgres RLS (`is_admin()` SECURITY DEFINER policies), `getAdminContext()` re-checked in every admin page/action/route, and middleware route protection. Never trust the client for role or ID.
- **Enrollment:** beta self-enroll (RLS allows insert only into published + `beta_open` courses, own user_id, status `active`). Payments will replace this.
- **Video:** R2 objects are reachable only through `/api/media/[lessonId]` ‚Äî authenticated + (enrolled OR admin), `Cache-Control: private, no-store`. Every response is a **bounded 4 MB `206` window** (HEAD for size ‚Üí clamped Range read), never the whole file ‚Äî this is the fix for the intermittent Cloudflare **Error 1102** (a full-file response + OpenNext's backpressure-less stream wrapper piled bytes into the 128 MB isolate). Upload is admin-gated R2 multipart (32 MiB chunks, key-prefix validated, replaced objects deleted) and **auto-compresses client-side before upload** ‚Äî WebCodecs via `mediabunny` (dynamically imported), hardware H.264 + faststart, with a bulletproof raw-upload fallback (`lib/video-compress.ts`, `components/admin/VideoUploader.tsx`); zero server cost, no CSP change. **Full analysis, compression details, scalability limits, and the Cloudflare Stream migration path: `docs/video-delivery.md`.** (Note: `is_free_preview` is defined but not yet enforced ‚Äî see that doc's tech-debt list.)
- **LGPD:** minimum collection, privacy page, platform noindexed. Never log PII, tokens, or secrets.
